## 📌 Project Overview
This project implements a three-node communication system based on the **Motorola 68000** and **PIA 6821** parallel interfaces.

The system involves three interconnected nodes:
- **Node B**, which transmits a predefined byte sequence
- **Node C**, which transmits a predefined byte sequence
- **Node A**, which receives data from both nodes through **interrupt service routines (ISRs)**

Node A collects incoming data as **3-byte messages**, alternates reception between B and C, stores the received bytes into a shared buffer, and applies a phase-based stop condition.

## 🧠 System Architecture
The project is organized around three communicating nodes:

- **Node A**: main receiver and controller
- **Node B**: transmitting node
- **Node C**: transmitting node

Node A uses two PIAs to handle communication with B and C and manages:
- interrupt recognition
- turn-based reception
- message reconstruction
- shared buffer storage
- message counting
- stop-condition verification

## ⚙️ Communication Logic
The system is based on an **interrupt-driven communication mechanism** with turn-based synchronization between the transmitting nodes.

### 🔄 Main Features
- Reception of **3-byte messages**
- Alternating handling of node B and node C
- Shared buffer management in node A
- Separate ISRs for the two input sources
- Phase-based execution logic
- Termination based on message content

## 📥 Data Transmission
Nodes **B** and **C** transmit predefined byte sequences through their PIA interfaces.

Node A:
- detects interrupts generated by incoming data
- reads bytes from the corresponding PIA
- stores them in a buffer
- reconstructs messages of length **3 bytes**
- alternates service between B and C through a turn variable
- tracks completed messages to control the execution flow

## 🧩 Interrupt Handling
The core of the system is based on two interrupt service routines:
- `ISR_B`, which handles data coming from **Node B**
- `ISR_C`, which handles data coming from **Node C**

These routines manage:
- turn assignment
- byte acquisition
- message completion tracking
- phase transition control
- stop-condition checks

## 📊 Phase Logic
Node A uses two execution phases:
- an initial phase for normal message reception
- a second phase activated after **6 received messages**

During the second phase, the system checks the first byte of completed messages from B and C.  
The communication stops when both corresponding messages begin with `0`.
