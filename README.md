## 📌 Project Overview
This project implements a three-node communication system based on the **Motorola 68000** and **PIA 6821** parallel interfaces.

The system simulates interrupt-driven data exchange between interconnected nodes, where:
- **Node B** sends fixed-length messages
- **Node C** sends fixed-length messages
- **Node A** receives data from both nodes through **interrupt service routines (ISRs)**

The communication flow is coordinated through a **handshake protocol**, ensuring proper synchronization between transmission and reception.

## 🧠 System Architecture
The project is organized around three communicating nodes:

- **Node A**: main receiver and controller
- **Node B**: sender node
- **Node C**: sender node

Node A handles incoming data from B and C through two PIAs and manages:
- interrupt recognition
- message reception
- turn alternation between senders
- buffer storage
- stop-condition checks

## 🛠 Tools & Technologies Used
- **Motorola 68000 Assembly**
- **PIA 6821**
- **ASIM / simulation environment**
- Memory and configuration files for node setup and execution

## ⚙️ Communication Logic
The system uses an **interrupt-driven handshake mechanism** to regulate message exchange.

### 🔄 Main Features
- Fixed-length message transfer
- Alternating reception from node B and node C
- Buffer-based storage in node A
- Synchronization through control signals
- Interrupt service routines for asynchronous communication
- Termination logic based on message content

## 📥 Message Transmission
Nodes **B** and **C** send predefined byte sequences through their PIA interfaces.

Node A:
- detects interrupts generated by incoming data
- reads bytes from the appropriate interface
- stores them in a shared buffer
- counts received characters and completed messages
- switches turn between B and C according to the communication protocol

## 🧩 Interrupt Handling
The core of the system is based on **interrupt service routines (ISRs)**:
- one routine handles messages coming from **Node B**
- one routine handles messages coming from **Node C**

These routines manage:
- sender access coordination
- lock and turn logic
- message completion tracking
- stop condition verification

## 📊 Simulation Goal
The goal of the project is to validate a low-level communication protocol in a multi-node system, showing how interrupt-driven coordination can be used to manage concurrent message sources in an embedded environment.

## 📌 Conclusion
This repository presents a low-level digital communication project that combines:
- **Motorola 68000 assembly programming**
- **parallel interface communication**
- **interrupt-driven synchronization**
- **handshake-based data exchange**

It is intended as an academic project focused on computer architecture, embedded systems, and low-level communication protocols.
